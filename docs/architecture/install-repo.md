## Overview

The "System Repo" install method is the recommended path for traditional Linux distributions (Ubuntu, Debian, Fedora). It hooks Antigravity into the OS's native package manager (`apt` or `dnf`), ensuring it receives automatic updates alongside normal system upgrades.

We use **Google Artifact Registry (AR)** as the backend hosting provider for both our APT and YUM/DNF repositories.

---

## Architecture: APT (Debian/Ubuntu)

The installer sets up a standard Debian repository architecture.

### Metadata Structure
APT relies on a strict hierarchy of cryptographic hashes to ensure the integrity of downloaded packages:
1. **`InRelease` / `Release`**: The master index file. It contains SHA-256 hashes of the architecture-specific `Packages` files. (Our repo uses `Release` + a detached `Release.gpg` signature).
2. **`Packages`**: The architecture-specific index (e.g., `binary-amd64/Packages.gz`). It contains metadata for every `.deb` available, including the SHA-256 hash of the actual `.deb` file.
3. **`.deb` Binary**: The actual payload located in the `pool/` directory.

### Client Configuration (`.list`)
```bash
deb [signed-by=/etc/apt/keyrings/antigravity-repo-key.gpg] https://us-central1-apt.pkg.dev/projects/antigravity-auto-updater-dev/ antigravity-debian main
```
- **`signed-by`**: This is a critical security feature. It instructs APT to verify the `Release` file signature *only* against our specific GPG key, preventing global "key confusion" attacks associated with the deprecated `apt-key` command.

*📚 Reference:* [Debian Repository Format Spec](https://wiki.debian.org/DebianRepository/Format)

## Architecture: DNF (Fedora/RHEL)

The installer configures a standard YUM/RPM repository architecture.

### Metadata Structure (`repodata/`)
DNF relies on XML metadata generated by the `createrepo_c` tool:
1. **`repomd.xml`**: The master index. It contains timestamps and checksums for the primary metadata files. DNF downloads this first to check if its local cache is stale.
2. **`primary.xml.gz`**: Contains essential package info, dependencies, and download paths.
3. **`filelists.xml.gz`**: A massive index of every file provided by the packages.

### Client Configuration (`.repo`)
```ini
[antigravity-rpm]
name=Antigravity RPM Repository
baseurl=https://us-central1-yum.pkg.dev/projects/antigravity-auto-updater-dev/antigravity-rpm
enabled=1
gpgcheck=0
```

> [!WARNING]
> **Security Gap (`gpgcheck=0`)**
> RPM packages use individual GPG signatures (verified via `gpgcheck=1`). Currently, Google Artifact Registry does not support signing the `.rpm` binaries during the automated CI/CD pipeline push. Therefore, `gpgcheck=0` is required. This means DNF accepts the package without cryptographic verification.

*📚 Reference:* [DNF Repository Configuration](https://dnf.readthedocs.io/en/latest/conf_ref.html#repo-options)

## Continuous Deployment via Artifact Registry

The `antigravity-auto-updater-dev` repositories are fully automated.

**The CI/CD Pipeline:**
1. A new tag is pushed to the main Antigravity repository.
2. The GitHub Action builds the `.deb` and `.rpm` packages.
3. The Action authenticates to Google Cloud via Workload Identity Federation (WIF).
4. Packages are uploaded using `gcloud artifacts apt upload` and `gcloud artifacts yum upload`.
5. Artifact Registry automatically rebuilds the APT `Release`/`Packages` indices and the DNF `repomd.xml`/`primary.xml.gz` files instantly.

This automated architecture means there is zero manual "repository management" (no need to manually run `dpkg-scanpackages` or `createrepo`).

## Rollback & Cleanup Mechanics

The installer implements defensive rollback logic.
- If `sudo apt install -y antigravity` fails, the script executes a `trap` that removes `/etc/apt/sources.list.d/antigravity.list`.
- This prevents a broken or unreachable repository from causing `apt update` to fail system-wide for the user in the future.

```bash
# Example Trap
trap 'rm -f /etc/apt/sources.list.d/antigravity.list' EXIT
```

## Essential Repository Skills

1. **`apt-get update -o Debug::Acquire::gpgv=true`**: Extremely useful for debugging GPG signature verification failures on Debian/Ubuntu.
2. **`dnf makecache --timer`**: Force DNF to refresh its local copy of `repomd.xml` without downloading the entire package database.
3. **`gcloud artifacts packages list --repository=antigravity-debian`**: Verify that the CI/CD pipeline successfully uploaded the latest `.deb` to Google Cloud.
